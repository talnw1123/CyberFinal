Reused Nonce Investigation

ทีม Blue Team ตรวจพบ traffic เข้ารหัสจากเครื่องที่ถูก compromise และพบข้อความ plaintext บางส่วนตกค้างอยู่ใน memory dump ของ endpoint เดียวกัน นักวิเคราะห์ต้องตรวจสอบว่า การเข้ารหัสถูกใช้อย่างปลอดภัยหรือไม่ และกู้คืนข้อความลับจากหลักฐานที่มีอยู่

แนวทางวิเคราะห์

  1. เปิด network_capture.json แล้วสังเกตว่า ciphertext ทั้งสองรายการใช้ algorithm และ nonce เดียวกัน
  2. AES-CTR สร้าง keystream จาก key และ nonce/counter แล้วนำมา XOR กับ plaintext
  3. เมื่อ reuse nonce กับ key เดิม จะได้ keystream เดิม ทำให้หา keystream ได้จาก ciphertext_a XOR leaked_plaintext
  4. นำ keystream ที่ได้ไป XOR กับ ciphertext_b เพื่อกู้คืน plaintext ชุดที่สอง
  5. อ่านข้อความที่ถอดได้และส่ง flag ในระบบ
