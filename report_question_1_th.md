# คำตอบข้อ 1 วิทยาการรหัสลับ

## 1.1 การออกแบบโจทย์หรือเครื่องมือฝึกปฏิบัติ Cryptography

ข้าพเจ้าออกแบบระบบเว็บ CTF Lab ชื่อ **KMUTT Crypto Incident Lab: Reused Nonce Investigation** โดยจำลองสถานการณ์ว่า KMUTT Cybersecurity Agency ตรวจพบ traffic เข้ารหัสจากเครื่องที่ถูก compromise และพบ plaintext บางส่วนตกค้างอยู่ใน memory dump ของ endpoint เดียวกัน ผู้เรียนต้องนำหลักฐานที่มีอยู่มาวิเคราะห์ข้อผิดพลาดในการใช้ AES-CTR และกู้คืนข้อความลับที่มี flag

ระบบถูกพัฒนาด้วย Flask และสามารถ deploy เป็น Web App บนคลาวด์ เช่น Render, Railway หรือ Fly.io ได้ ผู้ตรวจสามารถเข้าเว็บ อ่าน scenario ดาวน์โหลด artifact และส่ง flag เพื่อตรวจคำตอบได้ทันที

### วัตถุประสงค์

วัตถุประสงค์ของโจทย์นี้คือฝึกให้ผู้เรียนเข้าใจว่า cryptography ไม่ได้ปลอดภัยเพียงเพราะเลือก algorithm ที่แข็งแรง แต่ต้องใช้งานให้ถูกต้องด้วย โดยโจทย์เน้นช่องโหว่ **AES-CTR nonce reuse** ซึ่งเป็นกรณีที่ key และ nonce/counter เดิมถูกนำไปใช้เข้ารหัสข้อความมากกว่าหนึ่งชุด ทำให้เกิด keystream ซ้ำและสามารถนำ ciphertext กับ plaintext ที่ทราบบางส่วนมาใช้กู้คืนข้อความอื่นได้

### ความสำคัญและประโยชน์

โจทย์นี้มีความสำคัญต่อการเรียนรู้ด้านความมั่นคงปลอดภัยไซเบอร์เพราะเชื่อมโยง cryptography กับงาน Blue Team และ Digital Forensics โดยตรง ในสถานการณ์จริง นักวิเคราะห์อาจไม่ได้รับ key เข้ารหัส แต่มีหลักฐานจากหลายแหล่ง เช่น network capture, memory dump, log หรือไฟล์ชั่วคราว หากผู้เรียนเข้าใจหลักการของ stream cipher และ CTR mode จะสามารถประเมินได้ว่าการเข้ารหัสถูกใช้อย่างปลอดภัยหรือไม่

ประโยชน์ของระบบนี้คือช่วยให้ผู้เรียนฝึกคิดแบบ incident responder คือเริ่มจากการตรวจ artifact สังเกตความผิดปกติ ตั้งสมมติฐาน ทดลองวิเคราะห์ และยืนยันผลด้วย flag ระบบนี้ยังชี้ให้เห็นความสำคัญของ key management และ nonce management ซึ่งเป็นปัญหาที่พบได้จริงในการพัฒนา software และระบบสื่อสารเข้ารหัส

### รูปแบบข้อมูลนำเข้า ข้อมูลส่งออก และเงื่อนไขการใช้งาน

ข้อมูลนำเข้าที่ระบบให้ผู้เรียนดาวน์โหลดมี 3 ไฟล์ ได้แก่

1. `network_capture.json` มีข้อมูล algorithm, nonce และ ciphertext ของ encrypted note สองชุด
2. `leaked_plaintext.txt` เป็น plaintext ของ note A ที่ forensic team พบจาก endpoint memory
3. `solver_template.py` เป็นโครง script สำหรับช่วยทำ XOR และถอดข้อความ

ข้อมูลส่งออกที่ต้องได้คือ plaintext ของ encrypted note B และ flag ในรูปแบบ `KMUTT{...}` จากนั้นนำ flag ไปกรอกในหน้าเว็บเพื่อตรวจคำตอบ

เงื่อนไขของโจทย์คือผู้เรียนต้องใช้เฉพาะ artifact ที่ให้มา ไม่ต้อง brute force key และไม่ต้องโจมตีระบบภายนอก เพราะสาระสำคัญของโจทย์คือการวิเคราะห์ผลกระทบจากการใช้ AES-CTR nonce ซ้ำ

### ขั้นตอนการวิเคราะห์และการแก้ปัญหา

1. เปิดไฟล์ `network_capture.json` เพื่อตรวจข้อมูล ciphertext ทั้งสองชุด
2. สังเกตว่า encrypted note A และ encrypted note B ใช้ algorithm `AES-128-CTR` และใช้ `nonce_hex` ค่าเดียวกัน
3. เข้าใจว่า AES-CTR ทำงานโดยสร้าง keystream จาก key และ nonce/counter แล้วนำ keystream ไป XOR กับ plaintext เพื่อได้ ciphertext
4. เมื่อทราบ plaintext ของ note A จาก `leaked_plaintext.txt` และมี ciphertext ของ note A จึงคำนวณ keystream ได้จากสูตร `keystream = ciphertext_a XOR plaintext_a`
5. นำ keystream ที่ได้ไปถอด ciphertext ของ note B ด้วยสูตร `plaintext_b = ciphertext_b XOR keystream`
6. อ่าน plaintext ของ note B จะพบ flag คือ `KMUTT{CTR_NONCE_REUSE_EXPOSES_SECRET}`
7. นำ flag ไปส่งในหน้าเว็บ ระบบจะแจ้งว่าคำตอบถูกต้องและแสดง walkthrough

## 1.2 ความคิดเห็นเกี่ยวกับปัญหา อุปสรรค และข้อจำกัดของวิทยาการรหัสลับ

ในมุมมองของข้าพเจ้า ปัญหาสำคัญของวิทยาการรหัสลับในงานความมั่นคงปลอดภัยไซเบอร์ยุคปัจจุบันไม่ได้อยู่ที่ algorithm เพียงอย่างเดียว แต่อยู่ที่การนำไปใช้งานจริง หากเลือกใช้ algorithm ที่แข็งแรงแต่จัดการ key, nonce หรือ parameter ผิดพลาด ความปลอดภัยของระบบก็อาจพังได้ทันที เช่น AES-CTR หรือ AES-GCM ต้องห้าม reuse nonce กับ key เดิม เพราะอาจทำให้ข้อมูลรั่วหรือถูกปลอมแปลงได้

อีกปัญหาหนึ่งคือการจัดการกุญแจ ผู้ดูแลระบบต้องเก็บ key ให้ปลอดภัย หมุนเวียน key ตามรอบเวลา จำกัดสิทธิ์การเข้าถึง และมีขั้นตอน revoke key เมื่อเกิดเหตุรั่วไหล แต่ในทางปฏิบัติ key มักถูกฝังไว้ใน source code, config file หรือ environment ที่ควบคุมไม่ดี ทำให้ผู้โจมตีที่เข้าถึงเครื่องหรือ repository อาจนำ key ไปใช้ถอดข้อมูลได้

การเลือกใช้อัลกอริทึมและ mode ก็เป็นข้อจำกัดสำคัญ ผู้พัฒนาบางคนอาจเลือกใช้ mode ที่ไม่เหมาะสม เช่น ECB หรือใช้ library โดยไม่เข้าใจค่า default ทำให้เกิดช่องโหว่โดยไม่ตั้งใจ นอกจากนี้ผู้ใช้งานทั่วไปมักเข้าใจว่าเมื่อข้อมูลถูกเข้ารหัสแล้วจะปลอดภัยเสมอ ทั้งที่ความปลอดภัยขึ้นอยู่กับการ implement, การจัดการ key และ threat model ด้วย

ในด้านการสืบสวนและตรวจพิสูจน์ cryptography เป็นทั้งเครื่องมือป้องกันและอุปสรรค หากผู้โจมตีใช้การเข้ารหัสอย่างถูกต้อง เช่น encrypted C2, ransomware หรือ encrypted archive นักวิเคราะห์อาจไม่สามารถอ่านเนื้อหาได้โดยตรง ต้องพึ่งพาหลักฐานอื่น เช่น memory artifact, log, metadata, endpoint telemetry, key ที่หลงเหลือใน process หรือพฤติกรรมเครือข่ายแทน

สุดท้าย cryptography ยังอาจถูกนำไปใช้ผิดวัตถุประสงค์ เช่น ransomware ใช้เข้ารหัสไฟล์เหยื่อเพื่อเรียกค่าไถ่ หรือ malware ใช้เข้ารหัส traffic เพื่อหลบเลี่ยงการตรวจจับ ดังนั้นนักความมั่นคงปลอดภัยไซเบอร์จึงต้องเข้าใจทั้งด้านการป้องกันและด้านการสืบสวน เพื่อประเมินความเสี่ยงและตอบสนองเหตุการณ์ได้อย่างเหมาะสม
